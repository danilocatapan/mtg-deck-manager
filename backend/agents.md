# agents.md

## Regras de desenvolvimento

- Sempre usar Clean Code.
- Sempre criar testes unitários para novas regras de negócio e endpoints.
- Sempre adicionar logs relevantes de entrada, saída e erro.
- Sempre justificar decisões técnicas em PRs, documentação ou comentários objetivos.
- Não usar IA como lógica principal da aplicação.
- Separar responsabilidades entre controller, service e domain/DTOs.
- Sempre expor endpoints documentados com OpenAPI.
- Evitar métodos longos com mais de 20 linhas sempre que possível.
- Nunca confiar em input do usuário; validar e sanitizar antes de processar.
